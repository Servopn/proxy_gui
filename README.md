# Claude Proxy GUI

Windows 本地 Claude/Anthropic 兼容代理，提供系统托盘、日志窗口、渠道轮换、模型自动选择与运行统计。

服务默认只监听 `127.0.0.1:18081`，并将请求转发到讯飞 MaaS Anthropic 兼容接口。

## 构建 EXE

仓库中的 `claude.ico` 会同时作为 EXE 文件图标。运行：

```powershell
.\build.ps1
```

产物位于 `dist\ClaudeProxyGUI.exe`。运行时需将 `.env` 放在 EXE 同目录，密钥不会嵌入二进制。

## 界面功能

### 主窗口

| 区域 | 说明 |
|------|------|
| **统计卡片** | 显示总请求数、错误数、Input/Output Token（万单位）、渠道数、当前模式 |
| **控制栏** | 实时日志标题、强制ProxyAuto开关 |
| **日志区域** | 黑底彩色日志，支持整行/局部两种着色模式切换 |
| **底部按钮** | 清空日志、隐藏到托盘、着色切换、密钥管理、渠道状态、ProxyAuto、系统配置、退出 |

### 日志着色

底部按钮 **"着色:整行"** 点击切换：

| 模式 | 效果 |
|------|------|
| **整行变色** | ERR红色、OK绿色、503/retry橙色、info蓝色 |
| **局部高亮** | 模型名多色区分、HTTP状态码红色、token绿色 |

模型名使用 12 色池自动分配，相同模型名每次出现颜色一致。

### 强制 ProxyAuto 模式

主界面控制栏提供 **"强制ProxyAuto"** 复选框，持久化到 `.env`：

- **关闭**（默认）：客户端请求什么模型，就使用什么模型
- **开启**：无论客户端请求什么模型，都强制使用 ProxyAuto 模型池自动选择最优模型

### 密钥管理窗口

提供三个标签页：

| 标签页 | 功能 |
|--------|------|
| **当前已应用密钥** | 查看当前所有渠道的密钥预览状态 |
| **解密新密钥** | 使用密码解密新的渠道密钥，支持批量解密并保存到 `.env` |
| **一键测试所有密钥** | 并发测试所有渠道密钥的可用性，使用 Kimi-K2.6 模型 |

### 渠道状态窗口

显示各渠道的请求量、成功率、评分和状态，按成功率排名。

### ProxyAuto 设置窗口

- **模型状态标签页**：显示各模型评分和使用情况
- **模型池配置标签页**：配置参与自动选择的模型池（上移/下移/添加/移除/恢复默认/应用）
- 模型池配置持久化到 `.env`

### 系统配置窗口

可调整参数（修改后持久化到 `.env`）：

| 参数 | 说明 |
|------|------|
| 最大重试渠道数 | 单请求最大渠道尝试数 |
| 连接池大小 | 连接池上限 |
| 渠道评分预热请求数 | 达到此数量后启用评分 |
| 渠道最少样本数 | 单渠道最少样本后才参与评分 |
| 评分模式最低平均分 | 低于此值退回轮询 |
| 渠道冷却上限 | 超过此数量退回轮询 |
| 模型评分预热请求数 | 达到此数量后启用模型评分 |
| 模型最少样本数 | 模型最少样本后才参与评分 |
| 模型冷却时间(秒) | 连续错误后冷却时长 |

## 目录结构

```
proxy_gui/
├── claude_proxy_gui.py          # 薄入口
├── build.ps1                    # 打包脚本
├── claude_proxy/                # 源码包
│   ├── config.py                # 常量/环境加载/全局开关
│   ├── logger.py                # 日志队列
│   ├── connection_pool.py       # HTTPS 连接池
│   ├── stats.py                 # ChannelPool + ModelPool
│   ├── proxy.py                 # HTTP 代理转发
│   ├── tray.py                  # 系统托盘
│   ├── startup.py               # 启动锁/端口检测/开机启动
│   ├── app.py                   # 应用入口
│   └── gui/                     # GUI 子包
│       ├── main_window.py       # 主日志窗口
│       ├── key_manager.py       # 密钥管理
│       ├── channel_status.py    # 渠道状态
│       ├── proxy_auto.py        # ProxyAuto 设置
│       ├── config_window.py     # 系统配置
│       └── utils.py             # 工具函数
├── .env.example                 # 示例配置
└── claude.ico                   # 图标
```

## 主要行为

- 仅监听本机回环地址。
- 保留查询参数、JSON 与非 JSON 请求体，并支持 chunked 请求。
- 复用可用的上游 HTTPS 连接。
- 区分客户端请求数和上游重试次数。
- 网络密钥测试在后台运行，不阻塞 Tkinter 主线程。
- 退出时正常关闭托盘、HTTP 服务和连接池。
- 防止重复启动：已运行实例会自动激活已有窗口。
- 日志写入 EXE 目录下的 `logs/` 文件夹。
